# บันทึกการทำงาน TMS Pro

วันที่บันทึก: 29 เมษายน 2026
อัปเดตล่าสุด: 30 เมษายน 2026

## กติกาสำคัญ

- PostgreSQL ใช้สำหรับโหลด/อ่านข้อมูลเท่านั้น
- ห้ามเขียน, แก้ไข, ลบ, สร้างตาราง หรือเปลี่ยน schema ใด ๆ ใน PostgreSQL
- ข้อมูลจาก PostgreSQL ที่ต้องการแก้ไข ให้ sync/คัดลอกมาเก็บใน MySQL ก่อน แล้วแก้ไขเฉพาะใน MySQL
- ห้ามบันทึกรหัสผ่าน, token, connection string หรือข้อมูลลับลงไฟล์เอกสารนี้

## สถานะระบบปัจจุบัน

- แอปรันที่ `http://127.0.0.1:3000/index.html`
- เพิ่มหน้า Login แล้ว
- Login ใช้รหัสพนักงานจาก PostgreSQL ตาราง `security.employee`
- ช่องรหัสผ่านใช้รหัสพนักงานเดียวกันกับ username
- PostgreSQL ถูกตั้งเป็น read-only mode ในฝั่งแอป
- มี guard ใน `db/postgres.js` ให้รับเฉพาะคำสั่งอ่าน เช่น `SELECT`, `SHOW`, `EXPLAIN`
- ถ้าพยายามเขียน PostgreSQL จะถูกบล็อกด้วย `PG_READ_ONLY_GUARD`

## ฐานข้อมูล

### PostgreSQL

- ใช้เป็นแหล่งข้อมูลต้นฉบับ
- ตาราง employee ที่ใช้งานตอนนี้: `security.employee`
- รหัสพนักงาน: `employeecode`
- ชื่อพนักงาน: `fullname`
- branch: `branch`
- department: `dept_code`

### MySQL

- ใช้เป็นพื้นที่ทำงานสำหรับข้อมูลที่ต้องแก้ไข
- เพิ่มตาราง `editable_employees`
- เพิ่มตาราง `editable_cars`
- ข้อมูลทรัพย์สินรถจาก PostgreSQL ตาราง `asset.assetdt` ถูก sync มาไว้ใน `editable_cars` เพื่อใช้เลือกทะเบียนรถ โดย PostgreSQL ยังอ่านอย่างเดียว
- ระบบตั้งเวลา sync ข้อมูลทรัพย์สินรถซ้ำทุก 30 นาที
- เพิ่มตัวกรองกันทรัพย์สินกลุ่มรถเข็น/ผ้าคลุม/GPS/เครื่องชาร์จ/หลังคารถ/คอกเหล็กออกจาก dropdown เลือกรถ
- ฐานรถของ SRental ที่พบจริงคือ `srt_sitecontroldb`; หากตั้งค่าเป็น `srental_sitecontroldb` ระบบจะแปลงไปอ่าน `srt_sitecontroldb`
- พนักงานจาก PostgreSQL ถูก sync มาไว้ใน MySQL แล้ว
- จำนวนที่ sync ล่าสุด: ประมาณ 4056 รายการ

## งานที่ทำแล้ว

- ออกแบบหน้าแอปใหม่ให้ดูเรียบร้อยขึ้น
- เพิ่ม icon ในเมนู
- ปรับระบบค้นหาสถานที่/แผนที่หลายรอบ
- เพิ่ม token แผนที่ที่ผู้ใช้ให้มาไว้ในระบบก่อนหน้า
- แก้การบันทึกแผนเดินทางที่ขึ้น Internal Server Error
- ปรับราคาน้ำมันให้สัมพันธ์กับข้อมูลในหน้าแดชบอร์ด
- เพิ่มระบบใบส่งของ/printout ตามฟอร์แมต PDF อ้างอิง
- เพิ่มการเลือกบริษัท
- เพิ่ม mapping บริษัทกับฐานข้อมูล
- เพิ่มหน้า Login
- เชื่อม Login กับ employee ใน PostgreSQL
- เพิ่มหน้ากลางหลัง Login สำหรับเลือกบริษัทก่อนเข้าหน้าหลัก
- เพิ่มโลโก้บริษัทจาก PDF เดิมให้หน้าเลือกบริษัท, เมนูซ้าย, หน้าข้อมูลบริษัท และหัวเอกสารปริ้น
- เพิ่ม API ดึงราคาน้ำมันจาก Bangchak และเติมช่องราคาต่อหน่วยในหน้าวางแผนเดินทางอัตโนมัติ
- ล็อก PostgreSQL เป็น read-only
- สร้าง flow โหลดจาก PostgreSQL แล้วแก้ไขใน MySQL
- ปรับหน้า printout เมนูวางแผนการเดินทางเป็นฟอร์ม PETTY CASH SHEET แนวนอนตามแบบ AESCON
- ปรับเลขที่เอกสาร printout ให้ขึ้นต้นด้วย `MRPCS` ตามด้วยตัวย่อบริษัท
- เพิ่มหน้า “ตรวจสอบสถานะเอกสาร” ในเมนูแผนการเดินทาง เพื่อค้นหาเลข TRV และดูว่าเอกสารอนุมัติถึงขั้นตอนไหนแล้ว
- เพิ่มปุ่มปริ้นเอ้าท์ในหน้าตรวจสอบสถานะเอกสาร โดยดึงข้อมูลเอกสารที่เลือกมาสร้าง PETTY CASH SHEET ได้ทันที
- เพิ่มคอลัมน์ปุ่มพรีวิวแยกในรายการหัวหน้า, HR, MD และบัญชี เพื่อเปิด modal ดูรายละเอียดเอกสารก่อนอนุมัติหรือทำจ่าย
- เพิ่มการตั้งค่าสิทธิ์เมนูอนุมัติรายรหัสพนักงาน เก็บใน MySQL/local cache และซ่อนเมนูอนุมัติสำหรับผู้ที่ไม่ได้รับสิทธิ์
- เพิ่มการดึงทรัพย์สินที่เป็นรถจากฐานข้อมูล `sge_sitecontroldb`, `aes_sitecontroldb`, `srental_sitecontroldb` มาใช้ในช่องเลือกรถของหน้าจัดรถ

## จุดที่ควรมาต่อครั้งหน้า

- ระบุว่าต้องการดึงตาราง PostgreSQL อะไรเพิ่ม
- ระบุหน้าที่จะนำข้อมูลไปใช้ เช่น ใบส่งของ, แผนการเดินทาง, รถ, โครงการ, ลูกค้า, สินค้า
- สำหรับข้อมูลที่ต้องแก้ไข ให้สร้างตาราง MySQL editable แยกก่อน
- พิจารณาสร้าง database user ใหม่แบบ read-only จริงใน PostgreSQL cloud
- เปลี่ยนรหัสผ่านฐานข้อมูลที่เคยใช้ และจำกัด IP ที่เข้าฐานข้อมูลได้

## หมายเหตุความปลอดภัย

- เอกสารนี้ตั้งใจไม่เก็บ secret ใด ๆ
- หากต้องแชร์โปรเจกต์ ห้ามแชร์ไฟล์ `.env`
- ควรใช้ `.env.example` สำหรับตัวอย่างค่า config โดยไม่ใส่รหัสจริง
